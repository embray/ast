# AST Library — Developer Guide for Claude

## What is this?

AST is a C library for attaching and manipulating world coordinate systems
(WCS) in astronomical data. It supports FITS-WCS, celestial/spectral
coordinates, and graphical output. Documentation: [SUN/210](https://www.starlink.ac.uk/docs/sun210.htx/sun210.html) (guide),
[SUN/211](https://www.starlink.ac.uk/docs/sun211.htx/sun211.html) (class reference).

## Build systems

Two parallel build systems exist:

- **Starlink Autotools** (`./bootstrap && ./configure && make`) — requires
  Starlink `starconf` and buildsupport infrastructure.
- **CMake** (`cmake -B build && cmake --build build`) — standalone, no
  Starlink dependencies. Minimum CMake 3.24.

The CMake build is the one being actively developed. Use it for testing.

## Building and testing

For a developer-oriented build with extra warnings and sanitizers enabled:

```shell
cmake -B build-dev \
  -DCMAKE_BUILD_TYPE=Debug \
  -DAST_ENABLE_WARNINGS=ON \
  -DAST_ENABLE_SANITIZERS=ON
cmake --build build-dev
ctest --test-dir build-dev --output-on-failure
```

always make sure that any code you add does not trigger new warnings
and that it also passes the sanitization tests. The AST C source
is not warnings clean with full warnings enabled but your changes
should not add to the problem.

## Architecture

### Link-time plugin system

`libast` references error-reporting and graphics functions without defining
them. Satellite libraries provide implementations:

- `ast_err` — error reporting (calls `astPutErr`)
- `ast_grf_2.0`, `ast_grf_3.2`, `ast_grf_5.6`, `ast_grf3d` — graphics stubs

On macOS, `libast.dylib` uses `-undefined dynamic_lookup`. Executables must
link satellite libraries with **whole-archive** linkage or symbols won't
resolve at runtime.

### Key generated files

- `ast.h` — public header, generated by `makeh` Perl script via C preprocessor
- `ast_err.h` — error codes from `ast_err.msg`, generated by `cmake/gen_ast_err_h.cmake`
- `config.h` — feature detection results
- `AST_PAR` — Fortran include file with `AST__BAD` etc.

### Vendored dependencies

- `wcslib/` — vendored from wcslib v2.9 (~2002) with AST-specific modifications
  (ast prefix on symbols, dynamic p/p2 arrays, TPN projection). Do NOT try to
  upgrade to modern wcslib — the API has diverged significantly.
- `pal/`, `erfa/` — vendored astrometry libraries, compiled into libast via
  `palwrap.c` with symbol renaming through `pal2ast.h`/`erfa2ast.h`.
- `cminpack/` — vendored Levenberg-Marquardt solver.

## Test conversion effort

We are converting Fortran test programs in `ast_tester/` to C. See `PLAN.md`
for current status and priorities.

### Conversion patterns

- Use **public C API** (`ast.h`) only
- `astWatch(&status)` + `astOK` for error checking (no Starlink EMS)
- `astToString`/`astFromString` for checkdump round-trips (replaces Fortran
  COMMON-block channel source/sink callbacks)
- `astMalloc`/`astFree` instead of Fortran PSX
- Fortran `err_mark`/`err_rlse` simply omitted
- Fortran 1-based indices become C 0-based (e.g. `astGetCell`)
- Each converted test has a header comment documenting differences from the
  Fortran original

### Adding a new test

1. Create `ast_tester/testfoo.c`
2. Add `ast_add_test(testfoo)` to `ast_tester/CMakeLists.txt`
3. Update `PLAN.md` with the new test
4. Build and run: `cmake --build build && ctest --test-dir build -R testfoo --output-on-failure`

## Code style

- C99, no C++ features
- AST uses its own object system with reference counting (`astBegin`/`astEnd`)
- Error handling via integer status variable and `astOK` macro
- Function naming: `astFoo` (public macro), `astFoo_` (internal C function)
- No Starlink dependencies in CMake build path

## Important notes

- When making fixes to the files in src/ ensure that the history in the prologue
  is updated with the change.
- Keep `PLAN.md` updated as work progresses on test conversions
- `astEqual` segfaults on some classes (e.g. SpecFluxFrame) — avoid using it
  without testing first
- The `WHOLE_ARCHIVE` linking pattern in `ast_tester/CMakeLists.txt` is
  critical — without it, tests segfault on null function pointers
